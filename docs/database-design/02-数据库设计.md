# 数据库设计文档

## 一、设计原则

### 1.1 三范式遵循
- **第一范式（1NF）**：确保每列保持原子性，不可再分
- **第二范式（2NF）**：确保表中的每列都和主键相关
- **第三范式（3NF）**：确保每列都和主键直接相关，而非间接相关

### 1.2 设计规范
- 表名使用小写字母，单词间用下划线分隔
- 主键统一使用 BIGINT 类型，自增
- 时间字段使用 DATETIME 类型
- 状态字段使用 TINYINT 类型
- 所有表必须包含创建时间和更新时间字段
- 逻辑删除使用 is_deleted 字段（0-未删除，1-已删除）

## 三、数据表设计

### 3.1 用户相关表

#### 3.1.1 用户表 (sys_user)
| 字段名 | 数据类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|----------|------|----------|--------|------|
| user_id | BIGINT | 20 | 否 | 自增 | 主键 |
| username | VARCHAR | 50 | 否 | - | 用户名 |
| password | VARCHAR | 100 | 否 | - | 加密密码 |
| nickname | VARCHAR | 50 | 是 | NULL | 昵称 |
| email | VARCHAR | 100 | 是 | NULL | 邮箱 |
| phone | VARCHAR | 20 | 是 | NULL | 手机号 |
| avatar | VARCHAR | 255 | 是 | NULL | 头像URL |
| status | TINYINT | 1 | 否 | 1 | 状态(0-禁用,1-启用) |
| last_login_time | DATETIME | - | 是 | NULL | 最后登录时间 |
| create_time | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | - | 否 | CURRENT_TIMESTAMP | 更新时间 |
| is_deleted | TINYINT | 1 | 否 | 0 | 逻辑删除(0-否,1-是) |

**索引设计：**
- PRIMARY KEY (`user_id`)
- UNIQUE KEY `uk_username` (`username`)
- UNIQUE KEY `uk_email` (`email`)
- UNIQUE KEY `uk_phone` (`phone`)
- INDEX `idx_status` (`status`)

#### 3.1.2 角色表 (sys_role)
| 字段名 | 数据类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|----------|------|----------|--------|------|
| role_id | BIGINT | 20 | 否 | 自增 | 主键 |
| role_name | VARCHAR | 50 | 否 | - | 角色名称 |
| role_code | VARCHAR | 50 | 否 | - | 角色编码(USER/CINEMA_ADMIN/SYS_ADMIN) |
| description | VARCHAR | 255 | 是 | NULL | 角色描述 |
| status | TINYINT | 1 | 否 | 1 | 状态(0-禁用,1-启用) |
| create_time | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | - | 否 | CURRENT_TIMESTAMP | 更新时间 |
| is_deleted | TINYINT | 1 | 否 | 0 | 逻辑删除 |

**索引设计：**
- PRIMARY KEY (`role_id`)
- UNIQUE KEY `uk_role_code` (`role_code`)

#### 3.1.3 用户角色关联表 (sys_user_role)
| 字段名 | 数据类型 | 长度 | 是否为空 | 默认值 | 说明 |
|--------|----------|------|----------|--------|------|
| id | BIGINT | 20 | 否 | 自增 | 主键 |
| user_id | BIGINT | 20 | 否 | - | 用户ID |
| role_id | BIGINT | 20 | 否 | - | 角色ID |
| create_time | DATETIME | - | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引设计：**
- PRIMARY KEY (`id`)
- UNIQUE KEY `uk_user_role` (`user_id`, `role_id`)
- INDEX `idx_user_id` (`user_id`)
- INDEX `idx_role_id` (`role_id`)
